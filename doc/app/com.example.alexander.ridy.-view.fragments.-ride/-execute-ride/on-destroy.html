onDestroy

fun onDestroy(): Unit

onDestroy() is called before the activity is destroyed. The system invokes this callback either because: -the activity is finishing (due to the user completely dismissing the activity or due to finish() being called on the activity), or -the system is temporarily destroying the activity due to a configuration change (such as device rotation or multi-window mode) The mMap has to be destroyed
